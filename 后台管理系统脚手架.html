📢 项目简介 ​

  • 🚀 开发框架： 基于JDK17、SpringBoot3、SpringSecurity6、JWT、Redis、Mybatis-Plus、Knife4j构建前后端分离，接口采用 RESTful 风格

  • 🔐 安全认证： 权限认证使用SpringSecurity、JWT，支持多终端认证，支持分布式、支持多数据源

  • 🔑 权限管理： 基于 RBAC 模型，实现细粒度的权限控制，涵盖接口方法和按钮级别。支持动态加载权限菜单，多方式轻松控制权限

  • 🌺 功能模块： 包括用户管理、角色管理、菜单管理、部门管理、字典管理等多个功能。

  • 🚀 兼容性： 用户管理默认兼容了mysql、达梦、人大金仓等数据库的sql写法，开箱即用（待完善）。支持打包 jar 或 war。

  • 📘 接口文档： 自动生成接口文档，支持在线调试，提高开发效率。

  • 🌈 集成插件：

  • 🚀 辅助工具： 代码生成器MybatisX插件，一键直接在对应目录生成增删改查。

分布式|功能拆分部署 ​

如果有功能模块需要拆分部署的需求。例如邮件模块，需要单独部署，新建 cetc-mail模块，该模块只引入 cetc-auth 包即可，打包主项目 cetc-admin 时只依赖cetc-mail包即可，要求redis必需连接同一个地址，才可以通过token获取到当前用户。

项目结构约定 ​

  • jar 包的引入，统一在主pom中定义 version 并引入，其他子pom文件，禁止引入 version 。便于版本统一管理

  • mapper.xml 文件放在java包里对应的 Mapper类 同级目录下，方便功能迁移（一般都是放在resources下，考虑到功能拷贝，不需要再去过滤对应xml文件）。

    默认java目录不加载xml文件，故而，需要在主pom文件中，加入去掉过滤xml的配置

    xml
    <build>
    <!--保证mapper.xml被打包，否则无法识别xml-->
    <resources>
        <resource>
            <directory>src/main/java</directory>
            <includes>
                <include>**/*.xml</include>
            </includes>
            <filtering>false</filtering>
        </resource>
    </resources>
</build>

  • DO（Data Object）：阿里巴巴专指数据库表一一对应的 POJO 类。本项目采用此规则，例如 sys_user 表对应实体命名 UserDO。

  • Query：数据查询对象，各层接收上层的查询请求。注意超过 2 个参数的查询封装，禁止使用 Map 类来传输。本项目采用此规则，例如 sys_user 表对应实体查询命名 UserQuery。

  • 默认生成数据库主键为雪花算法，解决不同系统间数据交换自增主键冲突的问题。

  • 默认继承BaseDO的数据库表，必需包含id、create_time、update_time字段。

  • 持久层采用mybatis-plus框架，全局更新策略设置ALWAYS。这样能保证提交表单中空字段，值的更新成功，如果采用默认，非必填字段的空值无法更新成功，所以如果更新部分字段，不可直接使用service层中updateById方法，会将其他字段置空，要求强制手写更新xml语句。

    yaml
    mybatis-plus:
  global-config:
    db-config:
    	# 更新策略	
      update-strategy: always

java
package cn.com.cetc.core.config;

/**
 * mybatis-plus 配置类
 */
@Configuration
@EnableTransactionManagement
public class MybatisPlusConfig {

    /**
     * 分页插件
     */
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        return interceptor;
    }

}

自动填充实现 ​

本系统默认填充 create_time 和 update_time 数据库字段。如有需要，可自行扩展。

配置 ​

java
package cn.com.cetc.core.config;

/**
 * mybatis-plus 字段自动填充
 */
@Component
public class MybatisPlusMetaObjectHandler implements MetaObjectHandler {

    /**
     * 新增填充创建时间
     *
     * @param metaObject 元数据
     */
    @Override
    public void insertFill(MetaObject metaObject) {
        Date date = new Date();
        this.strictInsertFill(metaObject, "createTime", () -> date, Date.class);
        this.strictUpdateFill(metaObject, "updateTime", () -> date, Date.class);
    }

    /**
     * 更新填充更新时间
     *
     * @param metaObject 元数据
     */
    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", () -> new Date(), Date.class);
    }

}

联表查询 ​

NCIBase框架内置了pageVo 和 queryVo 接口，帮你写好了controller、service、mapper。只需要自己写对应的xml语句即可，适用于主表是当前DO表的sql语句，其他联表需自己重新定义接口。

联表分页POST：/pageVo

联表查询POST：/queryVo

  • xml

    分页page不用处理。查询条件取值一定注意以qe.xx方式，否则无法取到值。返回类型如下2种。

    • resultType 简单时使用，mybatis-plus 会自动转驼峰
    • resultMap 复杂或嵌套时使用
    xml
    <select id="pageVo" parameterType="cn.com.cetc.system.role.model.SysRoleQuery" resultType="cn.com.cetc.system.role.model.SysRoleDO">
    select
    t.id,
    t.name,
    t.code,
    a.label
    from sys_role t
    left join sys_role_user a on t.id = a.role_id
    <where>
        <if test="qe.code != null and qe.code != ''">
            and t.code = #{qe.code}
        </if>
    </where>
</select>

    /**
 * 字典转换converter
 * @author zhusenlin
 * @date 2024-07-06
 */
public class DictExcelConverter implements Converter<String>{

    @Override
    public Class<?> supportJavaTypeKey() {
        return String.class;
    }

    @Override
    public CellDataTypeEnum supportExcelTypeKey() {
        return CellDataTypeEnum.STRING;
    }

    @Override
    public String convertToJavaData(ReadCellData<?> cellData, ExcelContentProperty contentProperty, GlobalConfiguration globalConfiguration) throws Exception {
        return cellData.getStringValue();
    }

    @Override
    public WriteCellData<?> convertToExcelData(String value, ExcelContentProperty contentProperty, GlobalConfiguration globalConfiguration) throws Exception {
        ExcelProperty annotation = contentProperty.getField().getAnnotation(ExcelProperty.class);
        String typeCode = annotation.format();
        SysDictService dictService = SpringApplicationContextUtils.getBean(SysDictService.class);
        String label = dictService.getLabel(typeCode, value);
        return new WriteCellData<>(label);
    }
}

字典转换 ​

后端 redis 缓存：后端字典用redis缓存，命中则直接返回，否则去查询数据库，并且更新到缓存中，避免对数据库造成压力，能应对高并发。

前端缓存字典的问题

  • 字典必需永远和后端保持一致，维护难度大，稍不注意前后端就可能对不上
  • 虽然字典不常改但不代表永远不会修改，如果一有改动，数据就对不上

Java编码 ​

  • 除了 CRUD 及其他简单单表操作，其余均在 xml 文件中手写 sql，不允许在 Mapper 中使用 @select 注解和 mybatis-plus 的组合查询。原因，不利于理解和构造sql，无法直接复制完整语句，进行分析和后期sql优化。

  • xml文件中sql包含大于号”>“，小于号”<“等字符时，会影响xml的解析，不推荐使用转义字符&lt等，强制使用<![CDATA[]]>，原因：1、便于理解，一目了然。2、方便。便于直接将sql拷贝到执行器中，来回复制，无需再替换对应的字符。示例

    xml
    <select id="getZzjdId" resultType="java.lang.Long">
    <![CDATA[
        select id from t_yxdt_xlxx_zzjd where #{sjfssj} > kssj and #{sjfssj} < jssj and xl_id = #{xlxxId}
    ]]>
</select>

  • serviceImpl 中如果需要对 Mapper 接口的返回数据进行处理，则对应 Mapper 接口返回类型不能用 Map 接收，必需写Java实体类接收，避免硬编码，因sql语句变更，导致无法取值的bug。如果直接返回到前端，可酌情使用 Map（强烈建议不使用，达梦数据库，开发和线上尽管版本一致，但线上版本为客户统一设置，使用Map，可能会造成大小写不一致，导致赋值失败，产生bug）。

文件路径分隔符 ​

对获取的文件路径进行 split 分隔，为了兼容 linux 和 windows 应该利用正则表达式，不能直接使用/或\ \，否则无法解析正确

java
public class MyTest {

    private static final String separator = "/|\\\\";

    public static void main(String[] args) {
        File file = new File("D:\\test\\test1\\test1.txt");
        String absolutePath = file.getAbsolutePath();
        // Bad example
        // String[] split = absolutePath.split("\\\\");
        // Positive example：
        String[] split = absolutePath.split(separator); 
    }
}
同时更新主子表 ​

在已经存在的子表可能被其他表引用的情况下，使用如下方法，如果子表未被引用可以根据子表外键，先删除，后批量新增

java
/**
* 更新主子表，子表包含修改、新增、删除的子项
* @param ylBzl
* @return
*/
@Transactional(rollbackFor = Exception.class)
@Override
public Long updateData(YlBzlDO ylBzl) {
    // 更新主表
    getBaseMapper().updateById(ylBzl);
    // 处理子表
    List<YlBzlBzdDO> bzdList = ylBzl.getBzdList();
    // 过滤掉id为null的数据id集合
    List<Long> saveIds = bzdList.stream().filter(e -> e.getId() != null).map(e -> e.getId()).collect(Collectors.toList());

    LambdaQueryWrapper<YlBzlBzdDO> ew = new LambdaQueryWrapper<>();
    ew.select(YlBzlBzdDO::getId);
    Function<Object, Long> function = (e -> Long.valueOf(e.toString()));
    List<Long> dbIds = bzdService.listObjs(ew, function);

    // 1、新增；差集，求属于A不属于B的元素集合，则集合A-集合B
    List<YlBzlBzdDO> insertList = bzdList.stream().filter(e -> !dbIds.contains(e.getId())).collect(Collectors.toList());

    // 2、更新；交集，求属于A和属于B的元素集合，则A交B
    List<YlBzlBzdDO> updateList = bzdList.stream().filter(e -> dbIds.contains(e.getId())).collect(Collectors.toList());

    // 3、删除；差集，求属于B不属于A的元素集合，则集合B-集合A
    List<Long> deleteIds = dbIds.stream().filter(e -> !saveIds.contains(e)).collect(Collectors.toList());

    bzdService.saveBatch(insertList);
    bzdService.updateBatchById(updateList);
    bzdService.removeBatchByIds(deleteIds);
    return ylBzl.getId();
}

  • 取yml文件中的值，建议定义xxProperties类，使用@ConfigurationProperties(prefix = "spring.data.redis")注解，不建议直接使用@Value("${}")。原因：如果被多处引用，后期修改，需要全局替换，用类的话，全局维护一份，更利于维护。

URL命名规则 ​

  • 应使用连字符（-）来提高URL的可读性
    • 目录或文件名如果有2个单词组成时，一般建议中间使用中划线（-）隔开，不要使用下划线或者其他字符，在搜索引擎中，它是把中划线当做一个空格来处理，而下划线是被忽略的，例如seo-meihao会被读成seo与meihao，这是比较友好的写法。
  • URL路径中全都使用小写字母。原因：
    • 大小写混合容易出现拼写错误
    • robots区分大小写，可能会造成不收录。

Kafka ​

  • 分区数可以大于节点数，副本数不能大于节点数，因为副本需要分布在不同的节点上，才能达到备份的目的。

消息可靠性 ​

  • ack = 0，可靠性差，吞吐量最高。生产者将数据发送到网络数据流中的时候，Kafka就对当前的数据进行了响应（确认应答）；出现网络异常，会导致数据丢失。

  • ack = 1，可靠性高，吞吐量高。只完成了Leader副本的日志写入，当数据到达broker节点Leader副本时，进行响应；follower副本未写入，broker节点出现故障，也会导致数据丢失。

  • ack = -1（默认），可靠性最高，吞吐量差。会完成leader副本和follower副本的写入之后，再进行响应；需要注意，如果有5个副本，此时只有3个副本处于和Leader副本之间处于数据同步状态，那么

    此时分区就存在一个副本同步列表，我们称之为In Syn Replica，简称ISR。此时，Kafka只要保证ISR中所有的4个副本接收到了数据，就可以对数据进行响应了，无需等到全部副本收到数据。ISR副本

    数量约束可通过配置文件配置，参数名为 min.insync.replicas 默认值为1（不推荐），最小推荐值为2

幂等性 ​

  • enable.idempotence，默认false。只能做到单分区上幂等，多分区无法保证。

存储参数 ​

  • log.segment.bytes。存储段大小，默认1G
  • log.roll.hours。日志文件滚动间隔，默认7天

数据一致性 ​

设计目标是高吞吐、高并发、高性能，必需是分布式的。副本同步数据有延迟，如果挂掉，消费者消费的数据可能不一致，为了解决问题，引入了 高水位 （HW：Hight Watermark）机制，不同的副本之间维护一个水位线的机制（其实就是一个偏移量的概念），即 木桶理论。

消费者 ​

  • group_id。必需，消费组ID，用于标识完整的消费场景，一个组中可以包含多个不同的消费场景或对象。

FAQ ​

1、idea校验出现如下异常 ​

java: Generating equals/hashCode implementation but without a call to superclass, even though this class does not extend java.lang.Object. If this is intentional, add '@EqualsAndHashCode(callSuper=false)' to your type.

翻译：

**原因：**用 Lombok 的@Data注解的时候，当前类继承了一个公共的Base基础类。此异常就是父类的属性不会自动加上。

解决方案：

  • 当前类直接加入注解 @EqualsAndHashCode(callSuper = true)。

  • 在工程的根目录下新建 lombok.config 文件，内容如下：（推荐）

    sh
    config.stopbubbling=true
lombok.equalsandhashcode.callsuper=call

总结：

根据阿里 的编码规范，如果重写了 equals 必需重写 hashCode 方法。如果不重写可能会导致某些基于散列的数据结构（例如HashSet、HashMap等）出现问题。

专有名词解释 ​

  • DO（Data Object）：阿里巴巴专指数据库表一一对应的 POJO 类。此对象与数据库表结构一 一对应，通过 DAO 层向上传输数据源对象。

  • Query：数据查询对象，各层接收上层的查询请求。注意超过 2 个参数的查询封装，禁止使用 Map 类来传输。
