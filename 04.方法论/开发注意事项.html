    追加id排序

    1

  2. GROUP_CONCAT聚合

    语法：group_concat( [distinct] 要连接的字段 [order by 排序字段 asc/desc ] [separator '分隔符'] )。可以使用分隔符。

    一、报错原因

    1. 数据超出默认长度。通常是1024字节，当超出时，字符串被截断，导致数据不完整或报错。

      解决办法：修改配置文件group_concat_max_len = 102400，重启生效。

      查看：SHOW VARIABLES LIKE "group_concat_max_len";

    2. 排序和去重问题。不正确的使用ORDER BY和DISTINCT，可能导致结果顺序错乱或数据重复。

      解决办法：加上去重和排序。

      SELECT GROUP_CONCAT(DISTINCT column_name ORDER BY column_name)FROM table_nameGROUP BY some_column;

    二、用法

    1. 标准SQL中使用聚合函数，查询字段必须在group by 下显性显示。严格模式下

      sql
      select * from sys_user group by id #正确执行
select * from sys_user group by name # 报错this is incompatible with sql_mode=only_full_group_by
      1
      2

      解释：这是mysql对group by 对主键的豁免权。

      当使用主键（如id）作为分组字段时，由于主键的唯一性保证每组仅包含一条记录，此时SELECT *中的非聚合字段（如name）天然具有确定性值，因此MySQL允许省略这些字段的显式声明。这是对标准SQL的扩展优化‌

    public class BeanComparatorUtil {

    /**
     * 比较两个bean的字段，返回有变化的字段名和值
     * 如果无变化，则返回"内容无变动"
     *
     * @param oldObj
     * @param newObj
     * @return
     */
    public static <T> String compareField(T oldObj, T newObj) {
        StringBuilder sb = new StringBuilder();
        try {
            Field[] declaredFields = oldObj.getClass().getDeclaredFields();
            for (Field field : declaredFields) {
                if (field.isAnnotationPresent(CompareField.class)) {
                    String fieldName = field.getAnnotation(CompareField.class).value();
                    field.setAccessible(true);
                    Object oldValue = field.get(oldObj);
                    Object newValue = field.get(newObj);
                    // 恢复访问权限
                    field.setAccessible(false);
                    // 将其转为String
                    String oldValueStr = valueToString(oldValue);
                    String newValueStr = valueToString(newValue);
                    if (!oldValueStr.equals(newValueStr)) {
                        // 如果不相等，则记录，并判断是否为字典值
                        String readConverterExp = field.getAnnotation(CompareField.class)
                            .readConverterExp();
                        if (!readConverterExp.isBlank()) {
                            Map<String, String> map = parseDictData(readConverterExp);
                            oldValueStr = map.get(oldValueStr);
                            newValueStr = map.get(newValueStr);
                        }
                        // 格式：字段名: 旧值 -> 新值；
                        sb.append(fieldName).append(": ").append(oldValueStr)
                        .append(" -> ").append(newValueStr).append("；");
                    }
                }
            }
        } catch (IllegalAccessException e) {
            e.printStackTrace();
        }
        return sb.toString().length()  == 0 ? "内容无变动" : sb.toString();
    }


    /**
     * 适用于新增和删除
     * 包含字段注解的属性的值，并且有字典的，需要转换
     *
     * @param obj
     * @param <T>
     * @return
     */
    public static <T> String getCompareField(T obj) {
        StringBuilder sb = new StringBuilder();
        try {
            Field[] declaredFields = obj.getClass().getDeclaredFields();
            for (Field field : declaredFields) {
                if (field.isAnnotationPresent(CompareField.class)) {
                    String fieldName = field.getAnnotation(CompareField.class).value();
                    field.setAccessible(true);
                    Object value = field.get(obj);
                    field.setAccessible(false);
                    String str = valueToString(value);

                    String readConverterExp = field.getAnnotation(CompareField.class)
                    .readConverterExp();
                    if (!readConverterExp.isBlank()) {
                        // 字典处理
                        str = parseDictData(readConverterExp).get(str);
                    }
                    // 格式：字段名: 值；
                    sb.append(fieldName).append(": ").append(str).append("；");
                }
            }
        } catch (IllegalAccessException e) {
            e.printStackTrace();
        }
        return sb.toString();
    }

    /**
     * 解析字典数据字符串为 Map
     *
     * @param dictData
     * @return
     */
    private static Map<String, String> parseDictData(String dictData) {
        Map<String, String> map = new HashMap<>();
        // 字典处理。解析格式，转换为对应的label
        String[] split = dictData.split(",");
        for (String item : split) {
            String[] kv = item.trim().split("=");
            map.put(kv[0], kv[1]);
        }
        return map;
    }

    /**
     * 安全地将对象转为字符串表示，null 显示为 "-"
     */
    private static String valueToString(Object obj) {
        String result = "";
        if (obj == null) {

        }else if (obj instanceof String stringObj) {
            result = stringObj;
        } else if (obj instanceof BigDecimal bigDecimalObj) {
            // 去掉无效的0
            result = bigDecimalObj.stripTrailingZeros().toPlainString();
        } else {
            result = String.valueOf(obj);
        }
        if (result.isBlank()) {
            result = "-";
        }
        return result;
    }
}

导入 ​

  1. 批量批次导入时，需要加事务。

  2. 新增或更新数据时，先利用唯一标识进行查询数据库，区分哪些是新增的，哪些是更新的。查询条件可以用in，虽然受数据库max_allowed_packet参数影响，但是批量保存，最大是BATCH_COUNT=1000，所以此处可以用in。

  3. 批量新增时，如果数据量特别大，则考虑自己写saveBatch的xml。小数据量可以用mp的saveBatch，因为它是一条条执行，最后提交，同时要url连接要开启rewriteBatchedStatements=true参数。

  4. 批量更新时，建议查询出数据库对应的ID，调用mp的updateBatchById方法。定义一个唯一标识和ID的Map，然后取出对应数据库的ID，进行更新。

  5. 建议：导出结果展示错误列表、更新列表、新增列表。错误列表展示行、错误信息，其余只展示行。理由：如果返回excle对应的数据，a. 首先不利于页面展示通用性，每个页面都要写对应的字段名。b. 会增加返回json的大小，增加接口返回时间。c. excel的数据，用户是有的，对用户意义不大，只要对应行及错误信息即可。

数据库 ​

建表 ​

  1. 1对N的外键都建立普通索引。
  2. 1对1的外键都建立唯一索引。
  3. 必填字段设置不为空
  4. 状态字段都给默认值，并设置不为空
  5. 操作数据时，如果需要校验几个字段的唯一性，则可以存一个几个字段的md5值，用于唯一性判断，并添加唯一索引。
  6. 同一个条件下，多条数据的一条选择，可以利用规则组合，对其生成优先级的字段，越大优先级越高。

SQL书写 ​

  1. 分页查询时，order by 字段最后必须加上id asc排序，解决分页查询时同一个数据出现在不同分页。因为如果排序字段的值是相同的，第一页的数据有可能出现在第二页。
